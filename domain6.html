Domain 6 - Cryptography

Authenticating RAS (Remote Access Service) Clients

  • PAP (Password Authentication Protocol)
    • Used with PPP (Point to point Protocol) to authenticate clients
    • Sends password and pins in plain text
  • CHAP (Challenge Handshake Authentication Protocol)
    • Used with PPP to authenticate remote users
    • More secure than PAP
    • Client and server share a secret used in authentication (sent in cipher text)
    • Preformed at connection and several times after
    • MS-CHAP and MS-CHAPv2
      • Improved for MS clients
      • MS-CHAPv2 has the ability to preform mutual authentication
  • RADIUS
    • Use a centralized server to provide authentication for clients accessing multiple servers
    • Uses UDP
    • Only encrypts the password
  • Diameter
    • Improved version of RADIUS
    • Supports EAP (Extensible Authentication Protocol) to increase security
    • Uses TCP
  • XTACAS
    • Old Cisco proprietary protocol
  • TACAS+ (Terminal Access Controller Access-Control System Plus)
    • Cisco alternative to RADIUS
    • Can interact with Kerberos for authentication
  • AAA Protocols
    • Provide authentication, authorization, and accounting

Protocols

WiFi Security - TKIP vs CCMP

  • TKIP (Temporal Key Integrity Protocol)
    • Older encryption used with WPA
    • Didn't require new hardware
    • Has been cracked
  • CCMP
    • Used with WPA2
  • IEEE 802.1x
    • Server challenges systems trying to connect to authenticate
    • Prevents rogue devices from being able to access network resources
  • Personal vs Enterprise Mode
    • Personal mode lets users access the wireless network anonymously with PSK (preshared key) or passphrase
    • Enterprise mode forces users to authenticate with unique credentials before grantime them access to the network

Basic Cryptography

Hashing

Hashing Algorithms
  • Hashing Files
    • If a file's hash changes you know the integrity is lost but not why
  • Hashing Passwords
    • Passwords are often stored as hashes
    • Many systems have additional character salts to passwords to thwart attacks
  • Hashing Messages
    • Send a hash with a message to know if integrity is maintained
  • Using HMAC
    • Prevents an attacker from changing what the hash originally was before they modify a message

Symmetric Encryption

  • Uses same key to encrypt and decrypt data
  • Changes the key multiple times a day
  • Types:
    • AES
      • Block cipher
      • US block symmetric encryption standard
      • 128 bit keys but can also use 256 bits or 192 bits
    • DES (Data Encryption Standard)
      • Old
      • Encrypts data in 64 bit blocks
    • 3DES (Tripple DES)
      • Keep running data through a DES algorithm three times
      • Uses key sizes of 112 bits or 168 bits
    • RC4 (Rivest Cipher v4)
      • Symmetric stream cipher between 40 and 2048 bits
    • Blowfish
      • Strong symmetric block cipher
      • Encrypts in 64 bit blocks
      • Supports key sizes between 32 and 448 bits
      • Faster than AES except when changing keys/li>
    • Twofish
      • 128 bit
      • Built to compete with AES
      • 128 bit blocks
    • One Time Pad
      • Most secure
      • Physical card that is destroyed after use
  • Block Ciphers
    • Break data into specefic block size
    • Pads the data to fill the block if necesary
    • More secure than stream cipher
  • Stream Ciphers
    • Encrypts data as a stream of bits instead of dividing into blocks
    • More points to attack
    • Engine generating stream does not change

Asymmetric Encryption

  • Uses two keys (private and public)
  • Certificates
    • Digital document that includes public key information on the owner of the certificate
    • Issued and mangaged by Certificate Authorities
    • Parts:
      • Serial number (used to validate a certificate)
      • Issuer
      • Validity dates
      • Subject
      • Public key
      • Usage
  • RSA
    • Uses both public and private key in a matched pair
    • Commonly used to share a symmetric key
    • Uses prime numbers
    • Key size 1024 to 4096
  • Ephemeral Keys
    • Asymmetrical one time key used during exchange process
    • Allows for forward secrecy
      • Compromise of long term key does not compromise past session keys (perfect)
  • Elliptic Curve Cryptography (ECC)
    • Fast with small key sizes
    • Common in mobile computing
  • Diffie-Helman
    • Used to privately share a symmetric key between two parties
    • Asymmetric encryption mechanism

Other types of Cryptography

  • Steganography
    • Hides data inside data
    • Data can be hiden by maniplating bits or in the white space of a file
  • Quantum Cryptography
    • Use photons
    • Considered 100% secure
      • Once a baseline is established, the two ends can read changes in light patterns to know if someone is eavesdropping and cease transmissions

Using Crypto Protocols

Protecting Email

  • Digital Signatures
    • Definition
    • Proveds authentication, non-repudiation, and integrity
    • Process
      • Sender side:
        • Click send
        • Hash created
        • Hash is encrypted using senders private key
        • Sent to reciever
      • Recieving side: (verify the digital signature)
        • System retrives sender's certificate
        • Decrypts hash with the public key
        • Calculates new hash from the meassage
        • Compares the two hashes
  • Encrypting Email
    • Only Asymmetric Encryption
      • Sender finds reciever's public key and encrypts the email
      • Reciver decyrpts with their private key
    • Both Asymmetric and Symmetric Encryption
      • Use asymmetric encryption to privately send a symmetric key (think DH)
      • Symmetric encryption (faster) is used to encrypt the whole message
  • S/MIME (Secure/Multipurpose Internet Mail Extensions)
    • Most popular standards to digitally sign and encrypt email
    • Uses RSA (asymmetric) and AES (symmetric)
    • Can encrypt email at rest and in transit
  • PGP (Pretty Good Privacy) and GPG (GNU Privacy Guard)
    • Method to secure email communications
    • Create your own certificates
    • Uses decentralized web of trust
    • GPG is open source verison of PGP

Transport Encryption

Methods used to encrypt data in transit

  • IPsec
    • Encrypts data in tunnel mode with VPNs
  • Cipher Suites
    • Provide encryption, authentication, and integrity
    • Identified with cipher identifier (a string of hexadecimal character and a coded name)
    • Choosing the right ciphers to use depends on software and hardware restrictions

    • Key Stretching

      Helps prevent password cracking by using salting techniques
    • Bcrypt is based on the Blowfish block cipher (as much salt as you like)
    • PBKDF2 is sued by iOS, WPA2, and Cisco systems (set ammount of salt)

    In-band and Out-of-band Key Exchange

    • In-Band shares the key in the same communications channel as the encrypted data
    • Out-of-Band shares the key in a different communication channle as the encrypted data (ex physically delivering key)

    PKI (Public Key Infrastructure)

    Certificate Authority (CA)

    • Issues, manages, validates, and revokes certificates
    • Can be large/public or small/private

    Certificate Trust Paths and Trust Models

    • CAs are trusted by placing a root certificate (first certificate created by a CA) in a trusted root CA store

    Self Signed Certificates

    • Not trusted by default

    Wildcard Certificates

    • Certifies the subdomains of a site
    • Certificate for *.domain.com

    Registration

    • Steps:
      1. Create a public/private key pair
      2. Send a Certificate Signing Request (CSR) for the certificate

    Revoking Certificates

    • Reasons:
      • Key experation (most common)
      • Key compromise
      • CA compromise
      • Change of affiliation
      • Superseded
      • Cease of operation
      • Certificate hold
    • CAs use CRL (certificate revocation lists) to revoke certificates

    Validating Certificates

    • Client initiates session requiring certificate
    • Server responds with copy of ther certificate wth the public key
    • Client queries the CA for copy of the CRL
    • CA resonds with copy of CRL (if the certificate is revoked the client recieves an error)
    • Alternatively use the OCSP (Online Certificate Status Protocol) in which the CA simply responds with good, revoked, or unknown to the clients query using the certificate serial number

    Outdated Certificates

    • Usually when someone recieves a new smartcard
    • New certificate needs to be published to fix the problem

    Key Escrow

    • Useful for recovery

    Recovery Agent

    • Sometimes can be recovered from a private key escrow
