Domain 6 - Cryptography

Authenticating RAS (Remote Access Service) Clients

  • PAP (Password Authentication Protocol)
    • Used with PPP (Point to point Protocol) to authenticate clients
    • Sends password and pins in plain text
  • CHAP (Challenge Handshake Authentication Protocol)
    • Used with PPP to authenticate remote users
    • More secure than PAP
    • Client and server share a secret used in authentication (sent in cipher text)
    • Preformed at connection and several times after
    • MS-CHAP and MS-CHAPv2
      • Improved for MS clients
      • MS-CHAPv2 has the ability to preform mutual authentication
  • RADIUS
    • Use a centralized server to provide authentication for clients accessing multiple servers
    • Uses UDP
    • Only encrypts the password
  • Diameter
    • Improved version of RADIUS
    • Supports EAP (Extensible Authentication Protocol) to increase security
    • Uses TCP
  • XTACAS
    • Old Cisco proprietary protocol
  • TACAS+ (Terminal Access Controller Access-Control System Plus)
    • Cisco alternative to RADIUS
    • Can interact with Kerberos for authentication
  • AAA Protocols
    • Provide authentication, authorization, and accounting

Protocols

WiFi Security - TKIP vs CCMP

  • TKIP (Temporal Key Integrity Protocol)
    • Older encryption used with WPA
    • Didn't require new hardware
    • Has been cracked
  • CCMP
    • Used with WPA2
  • IEEE 802.1x
    • Server challenges systems trying to connect to authenticate
    • Prevents rogue devices from being able to access network resources
  • Personal vs Enterprise Mode
    • Personal mode lets users access the wireless network anonymously with PSK (preshared key) or passphrase
    • Enterprise mode forces users to authenticate with unique credentials before grantime them access to the network

Basic Cryptography

Hashing

Hashing Algorithms
  • Hashing Files
    • If a file's hash changes you know the integrity is lost but not why
  • Hashing Passwords
    • Passwords are often stored as hashes
    • Many systems have additional character salts to passwords to thwart attacks
  • Hashing Messages
    • Send a hash with a message to know if integrity is maintained
  • Using HMAC
    • Prevents an attacker from changing what the hash originally was before they modify a message

Symmetric Encryption

  • Uses same key to encrypt and decrypt data
    • <--->Skipped block vs Stream
  • Changes the key multiple times a day
  • Types:
    • AES
      • Block cipher
      • US block symmetric encryption standard
      • 128 bit keys but can also use 256 bits or 192 bits
    • DES (Data Encryption Standard)
      • Old
      • Encrypts data in 64 bit blocks
    • 3DES (Tripple DES)
      • Keep running data through a DES algorithm three times
      • Uses key sizes of 112 bits or 168 bits
    • RC4 (Rivest Cipher v4)
      • Symmetric stream cipher between 40 and 2048 bits
    • Blowfish
      • Strong symmetric block cipher
      • Encrypts in 64 bit blocks
      • Supports key sizes between 32 and 448 bits
      • Faster than AES except when changing keys/li>
    • Twofish
      • 128 bit
      • Built to compete with AES
      • 128 bit blocks
    • One Time Pad
      • Most secure
      • Physical card that is destroyed after use
  • Block Ciphers
    • Break data into specefic block size
    • Pads the data to fill the block if necesary
    • More secure than stream cipher
  • Stream Ciphers
    • Encrypts data as a stream of bits instead of dividing into blocks
    • More points to attack
    • Engine generating stream does not change

Asymmetric Encryption

  • Uses two keys (private and public)
  • Certificates
    • Digital document that includes public key information on the owner of the certificate
    • Issued and mangaged by Certificate Authorities
    • Parts:
      • Serial number (used to validate a certificate)
      • Issuer
      • Validity dates
      • Subject
      • Public key
      • Usage
  • RSA
    • Uses both public and private key in a matched pair
    • Commonly used to share a symmetric key
    • Uses prime numbers
    • Key size 1024 to 4096
  • Ephemeral Keys
    • Asymmetrical one time key used during exchange process
    • Allows for forward secrecy
      • Compromise of long term key does not compromise past session keys (perfect)
  • Elliptic Curve Cryptography (ECC)
    • Fast with small key sizes
    • Common in mobile computing
